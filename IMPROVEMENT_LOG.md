## 2026-07-05 — Iter.1

**診断要約（サイト全体レビュー）**

- 数式・定義の抜き取りチェック（MGF、フィッシャー情報量/CRLB、ガウス・マルコフ、ガンマ=χ²の特別形、
  マルコフ連鎖の定常分布 π₀=(1−b)/(2−a−b)、三大検定 LR/Wald/Score の数値）はいずれも整合。
- **可視化の設計欠陥を1件検出**: `topics/advanced2.js` の SVM デモが、決定境界を `y=3`
  （2クラスの平均中点）に**ハードコード**し、実際の最大マージン超平面を解いていなかった。
  本文・note は「サポートベクターが境界を決める／遠くの点を動かしても境界は不変」と説明するが、
  実装ではどの点を変えても境界は常に y=3 のままで、**学習目標（=SVが境界を規定する）を体験できない**。

**選んだ課題と理由**
- SVMデモの境界ハードコード（優先度: 内容の誤り > **誤解を招く可視化** > …）。
  「動かせるだけで学習目標に結びつかない」典型例であり、可視化中心という本サイトの中心思想に反するため
  最優先で修正。

**変更内容（差分は最小）**
- クラスは y 方向にのみ平均が異なり分散は等方的 → 最大マージンの分離方向は水平、という前提のもとで、
  境界を「A の最上点」と「B の最下点」（＝最も近い対向点＝サポートベクター）の**中点**に置くよう変更。
  マージン幅 = (botB − topA)。SVを丸で囲んで強調し、境界が**その2点だけ**で決まることを可視化。
- クラスが重なり線形分離できない場合（botB ≤ topA）は、境界をクラス平均中点にフォールバックし
  「線形分離できない → ソフトマージンが必要」と赤字表示。
- 散布の分散を sd=0.55 に、初期 gap を 3.4（既定で明確に分離）に調整。note を実挙動に合わせて書き換え。

**検証結果**
- `node --check topics/advanced2.js` OK、reload 後 console error 0。
- 境界値でのロジック検算（demo と同じ計算を再現）:
  - gap=0.5 → 非分離(soft margin), margin=0, NaNなし
  - gap=3.4(既定) → 分離, margin≈1.06 / gap=5 → margin≈2.66（隔たり↑でマージン↑を確認）
  - 再サンプルで境界が 2.34→2.86→3.07 と**データに応じて移動**＝サポートベクター依存を確認。
- スクリーンショットで境界・マージン帯・囲んだSV・「他点は無関係」注記の表示を目視確認。

**次回への申し送り**
