## 2026-07-05 — Iter.D1: カラーシステムの確立とグラフとの色分離

**改善テーマ**: デザインシステムをトークンとして確定し、UIアクセント色をグラフ系列色から分離する。

**変更内容**
- `DESIGN_SYSTEM.md` を新設。カラー／タイポ／余白／コンポーネントのトークンを一元定義。ダークモードは当面非対応と決定（理由も記載）。
- **UIアクセントを `#4f46e5`（インディゴ）→ `#3b4557`（低彩度スレート墨）に変更**。旧色はグラフ系列1 `#4f6df5`（青）とほぼ同一で、UIクロムとグラフが同化していた。
- 散在していたハードコード色を CSS 変数化: `--surface` `--muted-2` `--nav-ink` `--hover` `--warn-bg/line/bar` を新設し、サイドバー文字色・ホバー面・nav-group・例題ボックスをトークン参照に統一。
- 注記(`.note`)・まとめ(`.keypoints`)・クイズ解説(`.quiz-explain`)の背景を旧インディゴ面 `#eef0ff` ＋枠 `#dfe2fb` から無彩色 `--surface`＋`--line` に変更。
- `::selection` のインディゴを同スレートに合わせて変更。
- CSS変数のみの変更で、HTML構造・JS・グラフ描画コード（`app.js` の系列色）は一切触れていない。

**影響範囲**: 全ページの UI クロム（サイドバー選択・見出し `h3` の縦バー・カード左バー・スライダーつまみ・値バッジ・クイズ番号・ロードマップ badge・注記/まとめ/解説ボックス）。グラフ内部の描画色・数式・本文コンテンツは不変。

**before-afterで良くなった点**（1300px / 420px 実機スクショで確認）
- デモの青い棒（グラフ系列1）が**ページ唯一の青**として際立つようになった。従来は同色のインディゴがサイドバー・注記・見出しバーに散っており、グラフの色が「意味を持つ信号」として弱かった。
- ページ全体が低彩度スレート＋墨の無彩色基調になり、「落ち着いた学術的ミニマル」の方向に前進。装飾的な彩度を足さずに、グラフだけが色を持つ構図を実現。
- 色がトークン化され、以降の改善で参照一元管理が可能に。
- 数式（KaTeX）レンダリング・グラフの操作性・コントラストは不変。クイズ正誤色（teal/朱）も温存。ロードマップの+αトラックは朱(`--accent2`)を残し「範囲外」の区別を維持。

**次の改善候補**
1. **本文の読み幅（measure）制御**: `#main` の本文ブロックが広い画面で1行が長くなりがち。全角約40字を目安に読み幅上限を設ける（DESIGN_SYSTEM のレイアウト方針に沿う）。
2. **タイポグラフィ階層の微調整**: 見出しと本文の余白・行間の再点検（h3 直後の詰まり、リスト内の行間など）。
3. **注記/まとめ/例題ボックスの視覚的統一**: アイコン・左バー・余白のルールを揃え、種類の区別を彩度でなく形で表す。

## 2026-07-05 — Iter.D2: 本文の読み幅（measure）制御

**改善テーマ**: 本文プロースの1行の字数を抑えて可読性を上げる（図表・デモは全幅のまま）。

**変更内容**
- `--measure: 42em`（全角約42字）を新設。`.lead` と `.body` 直下のプロース（`p` / `ul` / `ol` / `h3` / `.note`）に `max-width: var(--measure)` を適用。
- **図表・デモ・数式ブロックは読み幅に縛らない**：`table.simple`（`.body` 直下でも `display:block` で全幅スクロール）、`section.demo` / `.example` / `.keypoints` / `.quiz`（`.body` の兄弟要素）はいずれも対象外＝全幅を維持。
- CSS変数＋子セレクタのみの変更。HTML構造・JS・グラフ描画には未介入。

**影響範囲**: トピックページの本文プロース幅のみ。デスクトップ幅で効き、モバイル（幅 < 42em）では従来どおり全幅。

**before-afterで良くなった点**（1360px / 420px 実機スクショで確認）
- 本文が端から端まで伸びず、約42字で折り返す読みやすい1カラムになった。視線の戻り幅が短くなり、情報密度の高い解説でも追いやすい。
- 一方で**前提条件の表・デモのグラフ・まとめ・クイズは全幅を維持**し、グラフが主役の構図は不変。テキスト列＜図表という自然な編集レイアウトに。
- 数式（KaTeX display）は読み幅の中で中央寄せ、崩れなし。モバイルは全幅のまま回帰なし。

**次の改善候補**
1. **注記/まとめ/例題ボックスの視覚的統一**: アイコン位置・左バー・余白のルールを揃え、種類の区別を彩度でなく形・ラベルで表す。
2. **タイポグラフィ階層の微調整**: h3 直後の余白、リスト行間、`.lead` のサイズ感の再点検。
3. **サイドバーの視線誘導**: グループ見出しと項目のインデント・現在地表示の強調度を調整。
